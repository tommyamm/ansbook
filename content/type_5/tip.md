

📋 Что проверяет это задание?

Согласно **официальному кодификатору ФИПИ**, задание проверяет:

> *Формальное исполнение простого алгоритма, записанного на естественном языке, или умение создавать линейный алгоритм для формального исполнителя с ограниченным набором команд, или умение восстанавливать исходные данные линейного алгоритма по результатам его работы.*

Чаще всего эти линейные алгоритмы основаны на **переводах чисел в разные системы счисления** и являются относительно простыми.

Рассмотрим перевод числа $123$ в $СС_2$ (систему счисления с основанием 2).

Программная реализация

```python
def perevod(n):
    r = ''  # результат
    while n > 0:
        r = str(n % 2) + r
        n = n // 2
    return r
```

Эта функция — **программная реализация ручного алгоритма**:
последовательно делим исходное число на основание новой системы счисления
и записываем остатки от деления в **обратном порядке**.

---

📐 Пошаговый разбор: $123_{10} \to СС_2$

| Шаг | Делимое | Частное | Остаток | Цифра (справа → влево) |
|:---:|:-------:|:-------:|:-------:|:----------------------:|
| 1   | 123     | 61      | **1**   | 1-я                    |
| 2   | 61      | 30      | **1**   | 2-я                    |
| 3   | 30      | 15      | **0**   | 3-я                    |
| 4   | 15      | 7       | **1**   | 4-я                    |
| 5   | 7       | 3       | **1**   | 5-я                    |
| 6   | 3       | 1       | **1**   | 6-я                    |
| 7   | 1       | **0**   | **1**   | 7-я                    |

Когда частное становится равным $0$ — процесс завершается.

Собираем остатки **снизу вверх** (в обратном порядке):

$$123_{10} = \underbrace{1\ 1\ 1\ 1\ 0\ 1\ 1}_{2} = 1111011_2$$

---

⚠️ Ограничение простой функции

Простая функция `perevod` **не всегда применима**.
Например, когда основание системы счисления **больше 10** — цифры числа
выходят за пределы десятичных символов `0–9` и обозначаются **буквами**:

| Десятичное | Шестнадцатеричное |
|:----------:|:-----------------:|
| 10         | A                 |
| 11         | B                 |
| 12         | C                 |
| 13         | D                 |
| 14         | E                 |
| 15         | F                 |

В таких случаях функцию необходимо расширить — использовать таблицу символов:

```python
DIGITS = '0123456789ABCDEFGHIJKLMNOPQRSTUVWXYZ'

def perevod(n, base):
    r = ''
    while n > 0:
        r = DIGITS[n % base] + r
        n = n // base
    return r
```

---

💡 Ключевые идеи для ЕГЭ

1. **Линейный алгоритм** — алгоритм без ветвлений, выполняющийся строго последовательно.
2. **Перевод числа** — деление с остатком, остатки читаются **снизу вверх**.
3. **Восстановление данных** — обратная задача: зная результат алгоритма, найти входные данные.
4. **Основание > 10** — нужна таблица символов (`A = 10`, `B = 11`, …).
